The Bio-AI Evidence Gap / 生物 AI 的证据缺口

单细胞 RNA 测序和 Lingshu-Cell 这类工作说明方向已经清楚：AI 正在从静态细胞快照走向可预测的虚拟细胞世界。高校和药企要采用这类系统，需要一条可复核的证据链：一次模型评测用了什么数据、什么版本、什么环境？一个候选靶点或药物为什么被推进？专家判断基于哪次预测？后续实验支持还是推翻了它？

Advaita + Setu 更务实的定位是：先做 Bio-AI evidence layer，把 benchmark receipt、model/data passport、专家审议和贡献结算做扎实；再在这个可信底座上承载 AI 找药、虚拟细胞和干湿实验闭环。

Core Insight: Setu Verifies the Work, Not the Biology / Setu 验证工作流，不替代生物学

The Four Pillars · 更贴合 Setu 的四大支柱

PILLAR 01

Verified Bio-AI Receipts

最务实的第一步是把 Bio-AI 的每次评测、推理和候选发现变成 receipt：模型版本、数据版本、评测脚本、seed、运行环境、输入输出摘要都被提交并锚定。这样可以服务论文复现、药企内部评测、合作方 benchmark 和后续候选发现。

PILLAR 02

Model / Data / Experiment Passport

Setu 不托管大模型权重和私有生物数据，而是记录它们的身份、版本、哈希、授权范围和收益分配逻辑。scGPT / Lingshu-Cell 方向模型、公开单细胞数据、合作实验数据、药企私有数据都可以进入 passport，真实数据留在链下，ownership 和后续贡献可被追踪。

PILLAR 03

Verified Scientific Deliberation

Setu 最有差异化的能力是处理“主观但必须可信”的科学判断：哪个候选值得推进、哪个解释更合理、一次失败实验是否推翻假设。Logic Clock 记录判断之间的因果关系；Sifakis 形式化验证用于检查关键时序与约束，让审议过程不是聊天记录，而是可检查的科学证据图。

PILLAR 04

Solver Execution & Settlement

Solver 节点可以执行可版本化的评测、推理、候选筛选和未来的 RL 微调任务。第一阶段重点是可审计执行记录；后续接入 TEE remote attestation + measurement allowlist 后，再升级为硬件级执行证明。Setu 原生结算用于分配模型调用费、数据收益、评测维护费和实验贡献费。

THE MAIN LOOP · DIRECTIONAL

Evidence-First Bio-AI Flywheel
先做可信证据层，再长出发现闭环

更务实的闭环从可复核证据开始：评测如何跑、数据从哪里来、专家为何认可、实验如何反馈。证据层一旦跑通，就能自然承载候选靶点筛选、虚拟细胞模型、私有测试集评测和后续干湿实验合作。

Setu 的价值是在这条链路里提供可信顺序和结算：哪一次任务先发生，哪个专家基于哪个结果推进，哪个实验验证或推翻了模型，哪份数据让模型变好。VLC 负责因果，Sifakis 形式化验证负责关键约束，Solver / TEE 路径逐步提供可验证执行，native settlement 负责把数据、算力、评测和实验贡献结算起来。

Where This Matters / 具体业务场景

Bio-AI Benchmark Receipt
近期：论文 / 模型 / 私有评测的可信凭证

研究者或药企提交模型、数据版本、评测脚本、seed、环境和输出摘要，生成可审计 receipt。先服务论文复现、模型比较、私有测试集评测和 BD 展示，这是最贴合 Setu 当前能力的入口。

Private Perturbation Evaluation
中期：药企 / 高校私有扰动评测

对 CRISPR、cytokine、药物扰动或候选靶点排序做私有评测。数据方不公开 hidden set，模型方提交任务，Setu 只记录可审计分数、模型 passport 和任务 receipt。

Virtual Cell Workflow
扩展：承载虚拟细胞模型，而不是夸口替代它

scGPT / Lingshu-Cell 方向模型作为外部或合作模型接入 Setu 工作流：细胞类型、个体身份、组织背景、干预条件、输出解释和专家审议都进入 evidence graph。

Discovery Triage
务实找药：先做人机共审候选排序

AI 可以给出候选靶点、候选分子或实验优先级，但推进与否仍由科学家判断。Setu 记录模型证据、专家意见、修订过程和后续结果，让“为什么推进这个候选”可追踪。

Dry-Wet Feedback & Settlement
长期：实验反馈与贡献结算

当合作实验室和数据源成熟后，wet-lab 结果可以链接回原始预测、专家审议和模型版本；数据贡献者、实验室、算力提供方和模型维护方通过 Setu receipt 自动分账。

Why Setu: Trust Layer, Not a Bio Model Claim

虚拟细胞模型来自 scGPT / Lingshu-Cell 这类 AI4SCI 模型和长期生物数据积累；Setu 的角色是可信底座：把模型评测、数据授权、专家审议、候选推进、实验反馈和收益分配串成可验证证据网络。

Advaita 可以在 Setu 上构建 Bio-AI 产品矩阵：模型层负责预测，科学家负责判断，实验负责验证，Setu 负责因果、证据和结算。这样既契合 Setu 当前客观能力，也能自然延展到 AI 找药、虚拟细胞和干湿实验闭环。

说明：receipt 的可信强度分阶段演进。第一阶段提供可追踪、可复核的因果记录；后续接入 TEE remote attestation 与 measurement allowlist 后，升级为硬件级执行证明。
